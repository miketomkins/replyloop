# Security Policy

## Responsible disclosure

Please report suspected vulnerabilities privately to the project maintainers through the repository security advisory workflow when available. If that is not available, open a minimal issue that requests a private contact path without including exploit details, credentials, personal data, phone numbers, chat IDs, or local paths.

## Public issue safety

Do not include secrets, real reminder content, personal identifiers, message transcripts, database files, logs, screenshots containing identifiers, or credentials in issues, pull requests, commits, or attachments. Use synthetic fixtures only.

## Supported baseline

The current repository baseline contains documentation and a public audit script. Runtime security guarantees apply only after implementation stages are completed and verified.
